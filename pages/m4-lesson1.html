Module 4 Lesson 1 Data Transfer Methods

Choosing a Transfer Method: SCP, SFTP, Globus, and the Scholarly Data Archive

Moving data onto, off of, and between IU's HPC systems is a routine part of research computing, and choosing the right tool for each situation saves time and avoids errors. The main options are SCP for quick one-shot copies between Linux hosts, SFTP for interactive browsing and small transfers, Globus for large files and cloud-connected workflows, and HSI or HTAR for working directly with the Scholarly Data Archive. Two hard constraints override all other considerations: protected health information (PHI) must always use Globus or SFTP, and HSI/HTAR are on-campus-only tools.

The slides below cover how to choose a transfer method, how each tool works, and how to archive files to the SDA correctly.

SCP (Secure Copy Protocol)

A command-line tool for copying files between two hosts in a single command. Best for quick, small copies when you already know the exact source and destination paths. Remote paths use the form username@hostname:path. No resume: a failed transfer restarts from the beginning.

SFTP (SSH File Transfer Protocol)

An interactive file transfer session over SSH. Lets you browse the remote directory tree before transferring. Best for small transfers when you need to navigate first. SFTP resumes interrupted transfers, unlike SCP.

SCP upload to Quartz

scp mydata.tar.gz username@quartz.uits.iu.edu:/N/slate/username/

SCP download from Quartz

scp username@quartz.uits.iu.edu:/N/slate/username/results.csv .

SFTP session commands

ls, cd, pwd navigate the remote side. put localfile uploads. get remotefile downloads. rename old new renames on the remote. lls, lcd, lpwd act on the local side. exit ends the session.

SFTP to the SDA

sftp username@sftp.sdarchive.iu.edu

Globus

A research data transfer service that schedules transfers between registered "endpoints." No local-vs-remote distinction: both sides are endpoints. Handles large files, cloud storage (OneDrive, Google Drive), and automatic retries. Access at https://globus.iu.edu/.

Globus endpoint naming at IU

IU HPC endpoints are named with the "IURT - " prefix. Examples: "IURT - Slate" (maps /N/project, /N/scratch, /N/slate), "IURT - Geode" (maps /N/u/username home directories), "IURT - Scholarly Data Archive", "IURT - OneDrive".

When to use Globus

Large files (over 1 GB) between IU systems; any transfer involving a personal computer; transfers to or from Google Drive or OneDrive; transfers to or from the SDA.

Globus Desktop client

Install the Globus Connect Personal client to make your local laptop or workstation a Globus endpoint, enabling direct transfers between your machine and any IU HPC system.

HSI (HPSS System Interface)

An interactive shell tool for accessing the Scholarly Data Archive (SDA). Load with module load hpss, then start a session with hsi. In-session commands mirror Unix: ls, cd, pwd, mkdir, put localfile, get remotefile, mv old new, exit. On-campus only; not PHI-safe.

HTAR

A tool that bundles many local files into a single .tar archive directly on the SDA, without creating an intermediate local archive. Best for large collections of small files, which the SDA handles poorly one-by-one. Load with module load hpss. On-campus only; not PHI-safe.

HTAR limits

Max individual file size: 68 GB. Max files per archive: 1 million. Max directory path: 154 characters. Max file name: 99 characters. These are more restrictive than the SDA itself.

HTAR create, list, extract

Create: htar -c -f my_archive.tar *. List contents: htar -tf my_archive.tar. Extract: htar -x -f my_archive.tar.

PHI transfer constraint

Protected health information (PHI) must use Globus or SFTP only, in any direction. HSI and HTAR do not encrypt data in transit and must never be used for PHI.

Off-campus SDA access

HSI and HTAR are on-campus tools. Accessing the SDA from off-campus requires either Globus/SFTP, or an exemption from store-admin@iu.edu to use HSI/HTAR.

SDA prep: small files

Tape handles many tiny files poorly. Bundle files under 1 MB into .tar or .zip archives before transferring to the SDA.

When to archive to the SDA

Archive at milestones: before submitting a paper, after a major analysis run, and at any significant phase change in your project.

  Did You Know?

Globus crossed the one-exabyte mark in 2020. That's one billion gigabytes moved across research networks worldwide. It took the service over 2,000 days to move the first 200 petabytes, but the most recent 200 petabytes took only 247 days.
