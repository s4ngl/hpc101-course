SSH, Modules, and SLURM

Connecting to IU's HPC systems and running work means three core skills: reaching the cluster over SSH, loading the software you need with the Lmod module system, and submitting jobs to the SLURM scheduler. Together these let you go from a login prompt to a running batch job.

The slides below walk through connecting, managing your software environment, and writing and submitting your first SLURM job.

 Try It Out

The terminal below runs a live SLURM simulator. The commands here match what you would run on Quartz or Big Red 200, so anything you practice transfers directly.

Explore the Queue

Get your bearings by checking what's running on the cluster and what resources are available.

squeue                          # Show all jobs in the queue
squeue --me                     # Show only your jobs
sinfo                           # Show partition and node status
sinfo -p debug                  # Show only the debug partition
scontrol show partition debug   # Show full details for the debug partition

Submit Your First Job

Create a simple batch script and submit it. Watch for the job ID returned by sbatch.

cat mpi_job.sh
sbatch mpi_job.sh
cat gpu_job.sh
sbatch gpu_job.sh
cat array_job.sh
sbatch array_job.sh

Monitor and Inspect

After submitting, track your job's status and inspect its details.

squeue --me                 # Check your job status
squeue --me --start         # See estimated start time if pending
scontrol show job <JOBID>    # Full details for your job
tail -f hello_<JOBID>.out    # Watch output live as job runs
sacct --me                  # Review completed job history

Control Your Jobs

Practice holding, releasing, and canceling jobs before they run.

sbatch --hold job.sh        # Submit in a held state
squeue --me                 # Confirm job is held (state = PD)
scontrol release <JOBID>     # Release it to run normally
scontrol hold <JOBID>        # Put it back on hold
scancel <JOBID>              # Cancel the job entirely
scancel --me                # Cancel all your jobs

Check Accounting

After jobs finish, review what resources they actually used.

sacct --me                                                      # Your recent jobs
sacct --me --format=JobID,JobName,State,Elapsed,MaxRSS,ExitCode # Detailed view
sacct --state=FAILED --me                                       # Any failed jobs
sshare                                                          # Your fairshare score

 Flashcards

SSH (Secure Shell)

A protocol for encrypted remote command-line access. Connect to Quartz with `ssh username@quartz.uits.iu.edu`.

Lmod

A software environment module system. Lets users load, swap, and unload software packages without system-level installs.

module avail

Lists all software modules available to load on the system.

module load

Activates a software package, making its executables and libraries available in your shell.

module purge

Unloads all currently loaded modules, returning the environment to a clean state.

module swap

Replaces one loaded module with another version, handling dependent module updates automatically.

module spider

Searches all available modules, including hidden ones. More thorough than `module avail`.

SLURM

The Simple Linux Utility for Resource Management. Schedules, runs, and tracks jobs on IU's HPC clusters.

SLURM partition

A logical grouping of compute nodes with defined resource limits and policies. Equivalent to a queue.

srun vs. sbatch

`srun` runs an interactive job that blocks your terminal. `sbatch` queues a script and returns your terminal immediately.

Wall time

The clock time a job is allowed to run. Set with `--time` in HH:MM:SS format. Not multiplied by core count.

%j in SLURM output filenames

Expands to the job ID at runtime. Use in `--output=job_%j.out` to give each job a unique output file.

squeue --me

Shows only jobs belonging to the current user.

scontrol show partition

Shows full partition configuration: default and maximum wall time, node list, and resource limits.

ThinLinc client

A desktop application for connecting to IU's Research Desktop with more reliability than the browser interface.

  Did You Know?

SLURM was originally developed at Lawrence Livermore National Laboratory and is now the most widely used workload manager in the world. The majority of the top 500 most powerful supercomputers on Earth use SLURM to schedule their jobs.
