Module 3 Lesson 2 Using IU HPC File Systems

Home Directories, File System Selection, Quotas, and the Scholarly Data Archive

IU's HPC file systems share a common logical structure but serve different purposes. Your home directory lives under /N/u/username/ with a subdirectory for each system you have an account on. The underlying physical paths are abstracted away through symbolic links, so what you see as /N/u/username/Quartz is actually rooted on the Geode 2 storage system.

The slides below cover how to navigate that structure, which file system to use at each stage of your work, how to check your storage usage, and how to move completed data to the Scholarly Data Archive.

 Flashcards

/N/u/username/ path structure

The parent path for all your HPC home directories. A subdirectory exists for each system you have an account on, e.g. /N/u/username/Quartz and /N/u/username/BigRed200.

Symbolic link (symlink)

A file that points to another path. IU uses symlinks to simplify home directory paths. /N/u/username resolves to a physical path on Geode 2 storage.

pwd vs. pwd -P

`pwd` and `pwd -L` return the logical path. `pwd -P` resolves all symlinks and returns the physical path on the underlying storage system.

HPC file system mount points

/N/slate (Slate personal), /N/scratch (Slate scratch), /N/project (Slate Project), /geode2/projects (Geode Project), /geode2/home (Geode home).

Redundancy vs. backup

RAID redundancy protects against drive failure. It does not protect against accidental deletion. None of the HPC file systems are backed up. If you delete a file, it is gone.

Where to run job I/O

Read and write job data to Lustre file systems (Slate Scratch or Slate). Reading and writing to your home directory during a SLURM job degrades performance for all users.

/tmp (local node disk)

Temporary storage local to the compute node. Faster than any networked file system. Data does not survive a reboot and is not visible from other nodes.

/N/scratch purge rule

Files not accessed in 30 days are deleted automatically. Archive to the SDA or move to Slate before that window closes.

Storage size decision guide

Under 1600 GB: use /N/slate. Between 1.6 TB and 15 TB: apply for Slate Project (free). Over 15 TB: see the IU Knowledge Base for Slate Project and Geode Project options.

Slate Project

Shared Lustre storage for research groups. Data is readable and writable by all project members. Up to 15 TB at no cost via an RT Projects request.

Geode Project

IBM Storage Scale storage for large shared datasets. An alternative to Slate Project for groups needing more than 15 TB or specific access patterns.

Inode limit

A cap on the total number of files and directories in a file system, tracked separately from storage size. Thousands of small files can exhaust inode limits before hitting the size quota.

quota command

A customized IU command loaded by default as a module. Shows size usage, size quota, inode usage, and inode quota across all file systems you have access to.

Scholarly Data Archive (SDA)

IU's long-term tape-and-disk archive based on HPSS. Holds approximately 79 petabytes, geographically mirrored between Bloomington and Indianapolis with dual copies on each tape system.

SDA access methods

Data is transferred to and from the SDA via hsi/htar command-line clients, SFTP, or Globus file transfer. The SDA does not mount as a file system.

  Did You Know?

The SDA stores data on physical tape cartridges, but retrieval is not as slow as you might expect. The HPSS software caches frequently accessed data on disk, so recently archived files often come back immediately without waiting for a tape mount.
