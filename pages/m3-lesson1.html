Module 3 Lesson 1 Understanding File Systems

Linux Storage Concepts, Parallel File Systems, Quotas, and Archiving

IU's HPC file systems build on standard Unix concepts (a tree-rooted directory structure, files represented as inodes, and data striped across redundant drives) but at a scale that spans thousands of physical drives across distributed storage servers. IU runs two primary storage stacks: IBM Storage Scale for home directories, and Lustre for the Slate family of file systems. Each is built for different access patterns, and using the wrong one for a workload degrades performance for everyone on the cluster.

The slides below cover how these systems work, where different types of data belong, and how to archive completed work to the Scholarly Data Archive.

 Flashcards

Inode

A data structure storing all file metadata (e.g. permissions, ownership, timestamps, and data block locations) but not the filename.

Why inodes don't store filenames

Filenames are stored in directory files as inode number mappings. This allows multiple names (hard links) to point to the same inode.

Hard link

A directory entry mapping a filename to an inode. Multiple hard links can reference the same file data.

File deletion in Unix

Removes the directory entry pointing to the inode. When the link count reaches zero, storage is freed. Unix calls this "unlinking."

GPFS (IBM Storage Scale)

IBM's General Parallel File System. IU uses it for Geode home directories and Geode Project storage.

Lustre

An open-source parallel file system for high-throughput I/O across many clients. IU uses it for Slate, Slate Scratch, and Slate Project.

RAID-6

A disk redundancy scheme using two independent parity sets distributed across drives. Tolerates up to two simultaneous drive failures.

df -Th

Lists all mounted file systems with type, size, used space, available space, and mount point.

Why to avoid ls -l on Lustre

Triggers a metadata lookup per file. On Lustre, metadata is stored separately and can bottleneck under load, causing delays on large directories.

Scholarly Data Archive (SDA)

IU's long-term tape-and-disk archive based on HPSS. Holds ~79 petabytes, mirrored between Bloomington and Indianapolis.

Globus

A research data transfer service for scheduling large transfers between registered endpoints. IU HPC systems and the SDA are both available as Globus endpoints.

/N/scratch (Slate Scratch)

High-speed temporary storage for active jobs. Files not accessed in 30 days are automatically deleted.

quota command

Shows storage usage and limits across all accessible file systems, including inode counts.

Metadata server

A dedicated server handling all file and directory metadata operations separately from data I/O. Excessive metadata requests bottleneck this server for all users.

  Did You Know?

IU's Slate Lustre file system spans over 3.5 petabytes of raw storage. Printed on standard paper at 80 characters per line, that much text would stack roughly 300,000 kilometers high, nearly the distance from Earth to the Moon.
