Module 1 Lesson 1 What is High Performance Computing?

Parallelism and Supercomputing Architecture

HPC systems are shared resources managed by research computing organizations, giving researchers access to computational power far beyond any personal machine. They are built from the same basic components as a workstation but are engineered at a completely different scale.

The core motivation behind supercomputing is parallelism. Breaking a problem into concurrent tasks lets it run faster, but every program has portions that must run sequentially. Even a small sequential fraction limits how much faster a program can run regardless of how many processors are added. The slides below cover these foundations.

 Flashcards

High Performance Computing (HPC)

Parallel processing across many compute nodes to solve problems too large or slow for a single workstation.

CPU (Central Processing Unit)

The primary processor in a computer, optimized for sequential instruction execution. HPC nodes typically have one or two multi-core CPUs.

Core

An independent processing unit within a CPU, capable of running its own thread simultaneously with other cores.

GPU (Graphics Processing Unit)

A processor with thousands of small cores designed for data-parallel computation. Used in HPC for scientific computing and AI workloads.

RAM (Random Access Memory)

Fast, volatile memory a CPU uses for active data. Jobs exceeding available RAM will fail or degrade due to swapping.

Parallelism

Breaking a computation into concurrent tasks that run simultaneously across multiple processors or cores.

Serial fraction

The portion of a program that must run sequentially. It limits total speedup regardless of how many processors are added.

Speedup

The ratio of single-processor runtime to multi-processor runtime. Decreases as the serial fraction grows.

Amdahl's Law

A formula that calculates maximum speedup given a program's serial fraction. Shows why even small serial portions dominate runtime at scale.

Compute node

A single server in an HPC cluster, typically with one or two CPUs, RAM, and sometimes GPUs.

Cluster

A set of compute nodes connected by a high-speed network and managed by a shared job scheduler.

Flynn's Taxonomy

A classification of computing architectures by instruction and data stream types: SISD (serial), SIMD (GPU-style), and MIMD (general parallel).

MIMD (Multiple Instruction, Multiple Data)

The parallel model used by most HPC clusters. Many processors run different instructions on different data simultaneously.

SIMD (Single Instruction, Multiple Data)

The parallel model used by GPUs. One instruction applied across a large data array simultaneously.

Embarrassingly parallel

A class of problems where tasks need no communication with each other. Trivial to parallelize.

Interconnect

The high-speed network linking compute nodes within a cluster. Low latency is critical for tightly coupled parallel jobs.

High Throughput Computing (HTC)

A computing model that maximizes the number of independent jobs completed over time, as opposed to HPC's focus on fast parallel execution of a single job.

petaFLOP

10^15 floating-point operations per second. Standard unit for measuring supercomputer performance.

  Did You Know?

Big Red 200 performs roughly 7 quadrillion floating-point operations per second. If every person on Earth did one math operation per second, it would take the entire global population over 850 years to match a single second of Big Red 200's output.
